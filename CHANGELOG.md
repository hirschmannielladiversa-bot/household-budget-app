## [2.5.0] — 2026-04-16

**レシート読み取りの PDF 対応 + data_editor 編集不可バグ修正**

### Added
- レシート読み取り機能が PDF ファイルに対応（サイドバー「📷 レシート自動読み取り」および支出管理タブ「📷 画像・PDF読み取り」両方）
  - `modules/receipt_reader.py`: `read_receipt(file_bytes, filename=None)` に拡張。拡張子とマジックバイト（`%PDF`）で PDF/画像を自動判定
  - PDF 処理は給与明細 PDF 読み取りと同じ二段構え: pdfplumber でテキスト抽出 → 50文字以下なら各ページを 400dpi で画像化し Gemini Vision にフォールバック
  - `app.py` の `file_uploader` に `pdf` 型を追加、PDF はファイル名表示に切替
- 使い方ガイド（`使い方ガイド.html`）に v2.5.0 新機能の視覚的ウォークスルーを追加
  - タブ配置・サブタブ配置・アップロードエリアの CSS モックアップ
  - 読み取り結果テーブルのサンプルイメージ
  - 対応 PDF の具体例（Amazon 注文確認、電子レシート、新幹線チケットなど）
- `requirements.txt` に `pdfplumber>=0.11.0` を明示追加

### Fixed
- `st.data_editor` の NumberColumn で `format="¥%,d"` を使っていたためフォーマットエラーで編集不可になるバグを修正
  - 根本原因: `%,d`（カンマ桁区切り）は sprintf 非対応。Streamlit のフォーマットパーサがエラーを出し data_editor 全体が read-only に落ちていた
  - 対処: `¥%,d` → `¥%d` 一括置換（3ファイル × 33箇所）
  - 実機検証済み（2026-04-16）

## [2.4.0] — 2026-04-16

**給与明細PDF読み取り：社会保険料の内訳展開**

### Added
- 給与明細PDF読み取り機能に、社会保険料の内訳（健康保険／厚生年金／雇用保険）を追加
  - Gemini プロンプトに `health_insurance` / `pension_insurance` / `employment_insurance` フィールドを追加
  - 読み取り結果テーブルの「社保」列を「健康保険」「厚生年金」「雇用保険」の3列に分割
  - 社保合計は3項目の和として自動算出（編集可能）
- `income_data[ym].deductions` に `health_insurance` / `pension_insurance` / `employment_insurance` キーを追加
  - 従来の `social_insurance`（合計値）は互換性のため残存

### Changed
- 手取り計算ルールを明文化: `手取り = 差引支給額 = 支給額合計 − (健康保険+厚生年金+雇用保険+所得税+住民税+他控除)`
- 既存 PDF フォーマット（内訳を返さないケース）との後方互換: `social_insurance` のみ取れた場合は健康保険欄に寄せ、ユーザが手動で配分可能

## [2.3.0] — 2026-04-15

**口座連携強化と多角的分析ダッシュボード**

### Added
- 支出タブ先頭に「🏦 口座の現在預金額」直接編集パネルを追加
  - 口座名・金融機関名・現在預金額を表形式で編集し一括保存
  - クレジットカード口座は除外表示
- 支出タブ「📄 データ一覧」に「口座」「口座処理済」列を追加
  - 登録済み銀行口座ラベルのプルダウン選択
  - 「🏦 口座から一括引き落とし」ボタン: 未処理行を残高から減算、処理済フラグで多重引き落とし防止
- 支出手入力フォームに引き落とし口座選択 + 即時残高反映オプション
- 収入管理タブに「✏️ 手入力」サブタブを新設
  - 年月・収入種別（給与/賞与/差額/その他）・金額・入金口座・メモ
  - 「選択口座の残高に自動加算」チェックで BankManager 経由の残高更新
  - 手入力履歴を `manual_notes` に蓄積し、既存データと共存
- 資産タブ「🏦 預貯金・口座」先頭に預貯金直接編集パネル
  - data_editor で口座名・金融機関・残高を編集、「💾 預貯金を保存」で永続化
  - 編集中合計 / 保存済み合計を並記
- DataLoader に `口座` / `口座処理済` 列を追加し、saved_expenses.csv で永続化
- 概要タブに「📈 多角的分析ダッシュボード」セクションを追加（6サブタブ）
  - 🎯 理想比率との乖離: カテゴリ別 実績 vs 理想比率を発散色ダイバージングバーで可視化
  - 💧 貯蓄率トレンド: 月次貯蓄率 + 3ヶ月移動平均 + 推奨20%基準ライン + 平均/黒字月/直近の指標
  - 🔥 月×カテゴリ ヒートマップ: YlOrRd で季節性・突発支出を直感把握
  - 📊 50/30/20 ルール: Needs/Wants/Savings の実績 vs 目標バー + 指標カード
  - 🌊 収支フロー Sankey: 手取り収入 → カテゴリ → 貯蓄・余剰 のフロー図
  - 📋 支出ランキング: 従来のカテゴリ円 + 高額支出一覧を温存

### Changed
- 年末調整セクションの月別給与データ表（12行 data_editor）を撤去し、**年間合計のみ** 入力フォーム（年間給与／年間賞与／源泉徴収税額／社会保険料）に刷新
- 源泉徴収票取り込み時の月別収入への推定リンク機能を撤去（年間データのみ保持）
- `show_bank_management_tab` の口座フィルタ selectbox を防御的 format_func に置換

### Fixed
- 口座残高エディタが空欄/NaN を書き戻して口座データを破損させる不具合を修正
- `accounts.csv` B005 の破損値（`None, None, 空`）を元の値（`ろうきん財形, 中央ろうきん, 180000`）に復元
- selectbox format_func が口座 name=None の場合に TypeError を発生させる問題を修正

### Verification

- Streamlit 起動確認（http://localhost:8502 で HTTP 200）
- 実データでの UI 動作確認は未検証

## [2.2.0] — 2026-04-13

**収入管理の口座一元化・PDF読み取り精度改善**

### Added
- 振込口座設定UI: BankManager口座とのドロップダウン連携（第1〜3口座）
  - 設定は user_settings.json に永続保存、入力し直さない限り引き継がれる
- 口座残高自動連携: 給与データ反映時にリンク先口座へ入金トランザクションを自動作成
- 収入管理サブタブ分離: 「PDF読み取り・サマリー」と「データ一覧・編集」
- 読み取り結果テーブルに第1口座・第2口座・財形の列を追加
- PDF複数ページ対応: Geminiプロンプトに JSON 配列出力指示を追加、配列パーサー実装
- 画像フォールバック解像度を 200→400 DPI に引き上げ

### Changed
- 手取り計算を「第1口座 + 第2口座 + 財形」に統一（読み取り結果・データ一覧の両テーブル）
- 口座名の取得元: Gemini結果 → 口座設定値を優先、Gemini結果をフォールバック

### Fixed
- 支出データ未登録時に収入タブ全体が非表示になるバグ（`if not months: return` を削除）
- 第2口座の保存条件バグ: `if _aname and _aamount > 0` → `if _aamount > 0`
  （振込先金融機関名が空のPDFで口座データがスキップされていた）

### Removed
- 口座からの給与自動検出機能（bank_manager トランザクション走査による検出を削除、手動口座設定に置換）

## [2.1.0] — 2026-04-12

**タブ構造の全面改修、PDF給与明細読み取り、Windows対応**

### Added
- タブ構造を8タブ→5タブに統合（概要 / 収入管理 / 支出管理 / 資産・税金・保険 / アドバイス）
- 給与明細PDF一括読み取り機能（複数ファイル対応、Gemini Vision API）
  - 書類種別の自動判定: 月例給与 / 賞与 / 差額 / 年末調整
  - 基本給・各種手当・控除内訳・振込口座の自動抽出
  - スキャンPDF対応（pdfplumber→画像化→Gemini Vision OCR）
- 収入データ一覧テーブル（data_editor で全項目編集可能）
- 支出管理タブ: レシート複数画像一括読み取り、CSV/Excel AI自動マッピング、手入力フォーム
- カテゴリ分類の銀行CSV文字正規化（小書きカナ→通常カナ、ハイフン→長音符、スペース除去）

- Windows版 .bat ランチャー（Shift-JIS/CRLF対応）

### Changed
- Gemini SDK を google-generativeai（サポート終了）から google-genai に移行
- bank_manager.reclassify_transactions を約1000倍高速化（パターン事前正規化 + ベクトル化）

- 概要タブの年間保険料: insurance_list から直接計算するように修正

- Streamlit初回起動時メールプロンプトのブロック（credentials.toml自動生成）
- requirements.txt 未検出エラー（フォルダ構成チェック追加）
- スキャンPDF読み取り失敗（PIL.Image.openがPDF非対応 → pdfplumber.to_image にフォールバック）
- 保険料が概要に反映されない（サイドバー削除後のリンク切れ修正）
- カテゴリ絞り込み中の保存で他データが消える（元index参照に修正）

### Removed
- グラフタブ（📈 グラフ）
- サイドバーの保険管理・金融資産管理UI（資産・税金・保険タブに統合）
