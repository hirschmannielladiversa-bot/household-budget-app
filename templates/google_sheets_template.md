# Google Sheets 家計簿テンプレート & NotebookLM 連携ガイド

このドキュメントでは、本アプリ（家計管理ダッシュボード）と相性の良い **Google Sheets 家計簿フォーマット** と、
NotebookLM を使った連携手順を説明します。

---

## 1. シート構成

### 支出シート（必須）

シート名: `支出` または `expenses`（自動検出対応）

| 列 | 見出し名 | 型 / 例 |
| --- | --- | --- |
| A | `日付` | 日付形式（例: `2024-03-01`） |
| B | `カテゴリ` | 下記カテゴリから選択 |
| C | `金額` | 数値（円、マイナスは使用しない想定） |
| D | `メモ` | 任意のテキスト |

### 収入シート（オプション）

シート名: `収入` または `income`（自動検出対応）

| 列 | 見出し名 | 型 / 例 |
| --- | --- | --- |
| A | `日付` | 日付形式（例: `2024-03-25`） |
| B | `項目` | 収入源（給与、副業、ボーナス等） |
| C | `金額` | 数値（円） |
| D | `メモ` | 任意のテキスト |

### 対応カテゴリ一覧（支出）

アプリ側のデフォルトカテゴリは次の通りです（`config/categories.yaml` と対応）。

- 食費
- 交通費
- 医療費
- 通信費
- 光熱費
- 住居費
- 保険料
- 娯楽費
- 教育費
- 日用品
- 衣服
- その他

カテゴリ名は **完全一致** が推奨です（例: 「食費（外食含む）」などと変形させない）。
どうしても増やしたい場合は `config/categories.yaml` を編集してください。

### 記入例（支出）

| 日付 | カテゴリ | 金額 | メモ |
| --- | --- | --- | --- |
| 2024-03-01 | 食費 | 3500 | スーパー買い物 |
| 2024-03-01 | 交通費 | 420 | バス |
| 2024-03-02 | 住居費 | 85000 | 家賃 |
| 2024-03-03 | 娯楽費 | 1200 | カフェ |

### 記入例（収入）

| 日付 | 項目 | 金額 | メモ |
| --- | --- | --- | --- |
| 2024-03-25 | 給与 | 280000 | 3月分給与 |
| 2024-03-15 | 副業 | 50000 | フリーランス案件 |
| 2024-06-25 | ボーナス | 500000 | 夏季賞与 |

ポイント:

- 日付は `YYYY-MM-DD` 形式が最も扱いやすいです。
- 金額は「円」単位の整数／小数どちらでも構いません。
- メモは空欄でも構いません（アプリ側で自動的に空文字扱いします）。

## 2. Google Sheets API 連携（推奨）

### セットアップ手順

1. **Google Cloud Console** (https://console.cloud.google.com/) にアクセス
2. プロジェクトを作成（または既存を選択）
3. 「APIとサービス」→「ライブラリ」→ **Google Sheets API** を有効化
4. 「APIとサービス」→「認証情報」→「サービスアカウント」を作成
5. サービスアカウントのキー（JSON）をダウンロード
6. JSONファイルを以下に配置:
   ```
   config/google_credentials.json
   ```
7. スプレッドシートをサービスアカウントのメールアドレスと共有

### 使用方法

1. アプリを起動
   ```bash
   cd "/Volumes/tkg SSD/household-budget-app"
   pip install -r requirements.txt
   streamlit run app.py
   ```

2. サイドバーの「📊 Google Sheets連携」セクションを開く

3. スプレッドシートのURLを入力

4. 「スプレッドシートから読み込み」をクリック

5. 支出・収入データが自動的に読み込まれます

### 自動検出される列名

アプリは以下の列名を自動的に認識します：

**支出シート:**
- 日付: `日付`, `date`, `Date`, `日時`
- カテゴリ: `カテゴリ`, `category`, `Category`, `項目`, `分類`
- 金額: `金額`, `amount`, `Amount`, `支出額`, `出金`
- メモ: `メモ`, `memo`, `Memo`, `備考`, `note`, `Note`

**収入シート:**
- 日付: `日付`, `date`, `Date`, `日時`, `入金日`
- 項目: `項目`, `収入源`, `source`, `Source`, `カテゴリ`, `種類`
- 金額: `金額`, `amount`, `Amount`, `収入額`, `入金額`
- メモ: `メモ`, `memo`, `Memo`, `備考`, `note`, `Note`

## 3. Google Sheets でのおすすめ設定

- **データ検証（プルダウン）**
  `カテゴリ` 列に「データの入力規則」を設定し、上記カテゴリ一覧から選択できるようにすると入力ミスを防げます。

- **フィルタビューの利用**
  月ごと / カテゴリごとにフィルタをかけて確認すると、NotebookLM なしでもある程度の分析が可能です。

- **1つのシートに継続記録**
  シートを月ごとに分けるのではなく、1つのシートに長期間記録していく方が、本アプリでのトレンド分析に適しています。

## 4. NotebookLM 連携の流れ

### 方法1: API連携（推奨）

1. 上記「Google Sheets API 連携」の手順でセットアップ
2. アプリのサイドバーからスプレッドシートURLを入力して読み込み
3. 「NotebookLM連携」セクションで「YAML生成」をクリック
4. 生成されたYAMLをNotebookLMのソースとして追加

### 方法2: CSV手動エクスポート

1. Google Sheets で家計簿シートを作成し、ある程度データを入力します。
2. 月末や分析したいタイミングで、Google Sheets のメニューから
   `ファイル` → `ダウンロード` → `カンマ区切りの値 (.csv)` を選択します。
3. ファイル名は `household_2024-03.csv` のように、年月が分かる名前にして保存します。
4. 本アプリの「📁 データ入力」からCSVをアップロード

### NotebookLMでの活用例

NotebookLM から自然言語で家計簿内容への質問ができます：

- 「先月と今月の食費の差を教えて」
- 「最近増えている支出カテゴリはある？」
- 「もし食費を1万円削ると、年間でどのくらい変わる？」
- 「ここ3ヶ月で一番増えた出費と、その理由を推測して」

## 5. NotebookLM × 本アプリの併用例

NotebookLM と本アプリは、次のように役割分担すると便利です。

- **NotebookLM**:
  - テキストベースでの「質問」「要約」「シナリオ検討」が得意
  - YAMLデータをソースとして追加することで、構造化されたデータを解析可能

- **本アプリ（Streamlit）**:
  - グラフでの視覚的な把握
  - カテゴリ別比率・月別推移・曜日×カテゴリのパターン分析
  - Claude を使ったファイナンシャルプランナー風アドバイス
  - 収支バランスの自動計算

おすすめの使い方:

1. Google SheetsにAPI連携で直接アクセス
2. 本アプリでグラフや統計値を確認
3. YAMLをエクスポートしてNotebookLMに追加
4. NotebookLMで「気になるポイント」を深掘り
5. アプリ内のAIアドバイスで具体的な行動プランを検討

## 6. トラブルシューティング

- **列名が認識されない**
  - 見出し行が 1 行目になっているか確認してください。
  - `日付`, `カテゴリ`, `金額`, `メモ` のスペースや全角・半角の違いがないか確認してください。

- **読み込み時に「必須列が見つかりません」と表示される**
  - CSV にエクスポートする際、別の列名に変更されていないか確認してください。
  - シート上でフィルタ行や余計なヘッダーを追加していないか確認してください。

- **カテゴリが意図しないグループにまとまる**
  - 「食 費」「食費（外食）」のように微妙に異なる名前が混在していないか確認してください。
  - 必要に応じて `config/categories.yaml` にカテゴリを追記し、アプリ側のカテゴリ定義を合わせてください。

- **Google Sheets API 接続エラー**
  - サービスアカウントのJSONファイルが `config/google_credentials.json` に配置されているか確認
  - スプレッドシートがサービスアカウントのメールアドレスと共有されているか確認
  - Google Sheets APIがGoogle Cloud Consoleで有効になっているか確認

- **収入シートが読み込まれない**
  - シート名が `収入` または `income` になっているか確認
  - シート選択で明示的に収入シートを指定してみてください

---

## 7. まとめ

- Google Sheets では **支出シート（4列）** と **収入シート（4列）** を作成
- **API連携** を使えば、CSVエクスポート不要で直接データ読み込み可能
- 読み込んだデータは:
  - グラフによる多角的な可視化
  - Claude / NotebookLM による言語的な洞察
  - 収支バランスの自動計算
  を両方活用できます。

家計簿のつけ方やカテゴリ設計で迷ったら、NotebookLM や本アプリ内のアドバイス機能に相談しつつ、
ご自身のライフスタイルに合った形に少しずつ調整していくのがおすすめです。
