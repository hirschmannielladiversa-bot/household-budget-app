# 家計管理ダッシュボード（macOS 配布版）

NotebookLM（Google Sheets）連携を想定した家計管理・分析アプリケーション。
Streamlitベースのダッシュボードで、多角的なグラフ分析とAIアドバイス機能を提供します。

**バージョン 2.5.1 / 配布最小版（Mac）**

> このフォルダは **配布最小版** です。個人データは含まれていません。
> 同梱のサンプルデータ（架空の「山田太郎」）で機能をお試しいただけます。

---

## まず最初に：ターミナルからアプリを導入する（初回のみ）

macOS では `.command` ファイルをダブルクリックや右クリック→「開く」で実行しようとすると、**Gatekeeper（セキュリティ機能）によりブロックされて正常に開けない**場合があります。
特に macOS Ventura 以降（Sequoia 含む）では、ダウンロードしたスクリプトの実行がより厳しく制限されています。

そのため、**初回は以下のターミナル手順で起動してください。**
一度ターミナルから実行すれば、2回目以降は `家計アプリ起動.command` のダブルクリックでも起動できるようになります。

### 手順（コピー＆ペーストでOK）

#### ステップ1: ターミナルを開く

- **Finder** → **アプリケーション** → **ユーティリティ** → **ターミナル.app** を開く
- または Spotlight（`Cmd + Space`）で「ターミナル」と入力して開く

#### ステップ2: アプリのフォルダに移動する

この配布フォルダを置いた場所に合わせて `cd` コマンドを実行します。

```bash
# 例: デスクトップに置いた場合
cd ~/Desktop/household-budget-app-v2.5.1-Mac

# 例: ダウンロードフォルダに展開した場合
cd ~/Downloads/household-budget-app-v2.5.1-Mac
```

> **ヒント**: `cd ` と打ったあとに、Finder からフォルダをターミナルにドラッグ＆ドロップすると、パスが自動で入力されます。

#### ステップ3: 必要なパッケージをインストールする（初回のみ）

```bash
pip3 install -r requirements.txt
```

> `externally-managed-environment` というエラーが出た場合:
> ```bash
> pip3 install --break-system-packages -r requirements.txt
> ```

#### ステップ4: アプリを起動する

```bash
python3 -m streamlit run app.py
```

ブラウザが自動で開き、家計ダッシュボードが表示されれば成功です。
開かない場合は http://localhost:8501 にアクセスしてください。

#### 終了するには

ターミナルで **`Ctrl + C`** を押す、またはターミナルウィンドウを閉じてください。

### 2回目以降の起動：ダブルクリック

初回の起動が成功したら、次回からは以下の方法でもOKです。

- `開始はこのフォルダ` 内の **「家計アプリ起動.command」** をダブルクリック
- 「開発元が未確認」と表示されたら → **右クリック** → **「開く」** → **「開く」**

## 事前準備：Python のインストール（初回のみ）

macOS には古い Python が入っている場合がありますが、**Python 3.12** のインストールを推奨します（3.13 / 3.14 は一部パッケージが未対応の可能性があります）。

### 方法1: python.org 公式インストーラー（推奨）

1. [https://www.python.org/downloads/macos/](https://www.python.org/downloads/macos/) を開く
2. 「macOS 64-bit universal2 installer」から **Python 3.12.x** をダウンロード
3. ダウンロードした `.pkg` をダブルクリックしてインストール

### 方法2: Homebrew（上級者向け）

```bash
brew install python@3.12
```

## 特徴

- **完全ローカル動作** — APIキーなしで全コア機能が使えます
- **AIアドバイス（オプション）** — Claude / Gemini APIで家計診断
- **銀行CSV / PDF / 画像取り込み** — 三菱UFJ、楽天銀行、ろうきん等に対応
- **レシート画像・PDF読み取り** — レシート写真やPDFから支出を自動入力（v2.5.0）
- **収入管理** — 月別入力 + 口座からの給与自動検出
- **ローン・負債管理** — 月々/賞与月返済、引落口座指定
- **セキュリティ** — API送信データ匿名化、入力検証、暗号化保存

## 初回の使い方（3ステップ）

1. **アプリを起動** → ブラウザに「ユーザーを選択してください」画面が出る
2. **「サンプル（山田太郎）」をクリック** → 架空のデータで全機能を体験
3. 機能に慣れたら → 「ユーザー切り替え」 → 「自分のデータ」を選んで、自分の家計を入力

> 「サンプル」を選んで編集しても、あなたの「自分のデータ」は別管理なので安心です。

## API設定（オプション）

| API | 用途 | 取得先 |
|-----|------|--------|
| Gemini API（無料枠あり） | レシート読み取り、PDF取り込み、アドバイス | aistudio.google.com |
| Claude API | 家計アドバイス | console.anthropic.com |
| Google Sheets API（無料） | スプレッドシート連携 | console.cloud.google.com |

APIキーはサイドバーに入力するか、環境変数で設定:

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
export GEMINI_API_KEY="..."
```

> ※ Gemini API は新 SDK `google-genai` を使用しています（旧 `google-generativeai` はサポート終了済み）。

## セキュリティ

- API送信データは匿名化（金額は概算、口座名/保険会社名は非送信）
- APIキーは画面に非表示、セッション終了で消去
- エラーメッセージは汎用化（内部情報を非表示）
- CSVインジェクション対策、HTMLエスケープ、入力検証

## トラブルシューティング

### 黒い画面に `command not found: python3` と出る
→ Python がインストールされていません。上記の「事前準備」を参照してください。

### `pip install` で失敗する（`externally-managed-environment` エラー）
→ macOS の新しい Python でよく出るエラーです。次のどちらかで対処:
1. `pip install --break-system-packages -r requirements.txt`
2. 仮想環境を作る: `python3 -m venv venv && source venv/bin/activate && pip install -r requirements.txt`

### Gemini API でエラーが出る
→ `google-genai` パッケージがインストールされているか確認: `pip3 show google-genai`
→ 入っていない場合: `pip3 install google-genai`

### ユーザー選択後の画面表示が遅い
→ 本バージョンで高速化済みです。1 秒以内に表示されるはずです。
もし遅い場合は `data/transactions.csv` に大量データがないか確認してください。
