Phase 14: レシート読み取り PDF 対応 + data_editor 編集不可バグ修正 (v2.5.0)

2026-04-16

写真しか扱えなかったレシート読み取りを電子レシート PDF にも広げ、同時に見えていなかった Streamlit の format 指定バグを潰したリリース。

📝 プロンプト #34 (レシート PDF 対応)
電子レシートや Amazon 注文確認の PDF を、レシート読み取りと同じ流れで取り込みたい。
✅ 結果
modules/receipt_reader.py の read_receipt(file_bytes, filename=None) を拡張。拡張子とマジックバイト (%PDF) で PDF / 画像を自動判定。
PDF 処理は給与明細 PDF と同じ二段構え: pdfplumber でテキスト抽出 → 50 文字以下なら各ページを 400dpi で画像化し Gemini Vision にフォールバック。
app.py の file_uploader に pdf 型追加、PDF はファイル名表示に切替。requirements.txt に pdfplumber を明示追加。
📝 プロンプト #35 (data_editor 編集不可バグ)
支出テーブルが読み取り専用になっていて編集できない。エラーは出ないけど入力できない。
✅ 結果
根本原因: NumberColumn(format="¥%,d") が Streamlit の sprintf パーサで拒否され、エラーは出さず data_editor 全体を read-only に落としていた (%,d は printf 非対応)。
対処: ¥%,d → ¥%d 一括置換 (3 ファイル × 33 箇所)。実機検証済み。
📝 プロンプト #36 (使い方ガイドの視覚化)
v2.5.0 の新機能をガイドに反映。どこのタブにアップローダーが増えたかを図で見せたい。
✅ 結果
使い方ガイド.html にタブ配置・アップロードエリア・読み取り結果テーブルの CSS モックアップを追加。対応 PDF の具体例 (電子レシート・注文確認・チケット) もリスト化。
機能追加 バグ修正

💡 この開発から学べること

プログラミング知識ゼロでもアプリが作れる

「〇〇を食費にして」「ローン管理を追加して」のような日本語だけで機能が実装された。

スクリーンショットが最強のバグ報告

「カテゴリが変わっていません」+スクリーンショットの組み合わせで、Claude Codeは問題を即座に特定できた。

セキュリティは後からでも対応できる

まず機能を作り、監査エージェントで問題を洗い出し、並列エージェントで一括修正するフローが効率的だった。

並列エージェントで作業速度が3倍に

セキュリティ修正を3体のエージェントに分担し、ファイル所有権を分けることで衝突なく並行作業できた。

キャッシュ問題は根気よく対処

CSVを書き換えてもStreamlitのsession_stateに古いデータが残る問題は、何度もやり取りして解決。「ブラウザをリロード」が最終手段。
